Architecture & Dataflow

从 LoRA 联邦长线实验到前端展示页面，数据是怎样流动的

这一页专门说明当前这套“注册表驱动 + 输出目录驱动”的论文原型系统架构。

系统分层

训练脚本层
run_centralized / run_sd15_lora_federated / run_config_queue
→
实验输出层
run_summary / metrics / samples / audit records
→
注册表层
portal_runs.yaml / comparison_groups.yaml / research_library.yaml
→
门户聚合层
repository.py 统一整理状态、指标、样图与论文说明
→
展示层
Flask 路由、模板页面、缩放预览和对照页

当前最重要的 5 个数据流步骤

1. 训练落盘

每组 100 轮长线实验把摘要、轮次指标、样图和审计结果写入对应输出目录。

2. 注册表登记

实验和对照页不再硬编码在 Python 字典里，而是通过 YAML 注册表统一声明。

3. 仓储层整理

repository.py 负责读取配置和输出目录，生成页面真正需要的高层字段。

4. 路由注入

app.py 把结果中心、专题页、研究页和详情页需要的数据注入模板。

5. 页面自动更新

即使某组实验还没跑完，门户也会先显示占位卡片；一旦指标或样图落盘，页面会自动补全。

扩展方式

后续加实验时要改哪些地方

  • 新增一组长线实验时，先补配置文件，再在 portal_runs.yaml 中登记标题、角色、输出目录和配置路径。
  • 新增一组论文主对照时，在 comparison_groups.yaml 里写入 comparison id 和对应 run_ids。
  • 新增参考文献、公式和中文解释时，只需要扩展 research_library.yaml。
  • 门户本身尽量只负责展示，不再把实验结论写死在模板里。
